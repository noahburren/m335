# Hybrid App – Steckbrief

## Kurzbeschreibung
Hybrid Apps sind Apps, die mit Webtechnologien wie HTML, CSS und JavaScript entwickelt und dann in eine native App-Hülle verpackt werden. Dadurch kann dieselbe Codebasis auf mehreren Plattformen wie iOS und Android verwendet werden.

---

## Typische Technologien

### Frameworks
- Ionic Framework
- Apache Cordova
- Capacitor

### Sprachen
- HTML
- CSS
- JavaScript
- TypeScript

### IDEs
- Visual Studio Code
- WebStorm

---

## Beispiele
- Sworkit
- Untappd
- MarketWatch

---

## Typische Anwendungsfälle
Hybrid Apps eignen sich besonders für:
- Firmen-Apps
- Content- und Informations-Apps
- einfache Shops oder Buchungssysteme
- Apps mit kleinem Budget
- Projekte, die schnell für mehrere Plattformen veröffentlicht werden sollen

---

# Eigenschaften

| Kriterium | Bewertung | Kurzkommentar |
|---|---|---|
| Platform APIs | ★★★★☆ | Zugriff auf Kamera, GPS usw. über Plugins möglich |
| Performance | ★★★☆☆ | Gut für normale Apps, aber schwächer als native Apps bei aufwendigen Animationen oder Games |
| Kosten / Zeit | ★★★★★ | Eine Codebasis für mehrere Plattformen spart Zeit und Geld |
| UX | ★★★☆☆ | Kann gut aussehen, fühlt sich aber manchmal weniger „native“ an |
| Offline | ★★★★☆ | Offline-Funktionen mit lokalem Speicher möglich |
| Distribution | ★★★★★ | Veröffentlichung über App Stores möglich |
| Updates | ★★★★★ | Updates oft schneller und einfacher |
| Plattformsupport | ★★★★★ | Läuft auf iOS, Android und teilweise auch im Browser |

---

# Vorteile
- Schnellere Entwicklung
- Günstiger als zwei separate native Apps
- Gemeinsame Codebasis
- Webentwickler können leichter Mobile Apps bauen

---

# Nachteile
- Weniger Performance als native Apps
- Manche Gerätefunktionen schwieriger nutzbar
- UI fühlt sich teilweise nicht komplett nativ an

---

# Merksatz
> „Hybrid Apps sind Webseiten in einer nativen App-Hülle.“
